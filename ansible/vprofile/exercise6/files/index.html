Subtopics
Playbooks and Modules — short, clear examples below.

Playbooks

Playbooks are YAML files that describe automation tasks as a series of plays. They define hosts, tasks and the modules those tasks use.

Example (simple playbook):
- name: Ensure Nginx is installed and running
  hosts: webservers
  become: yes
  tasks:
    - name: Install nginx
      apt:
        name: nginx
        state: present
    - name: Ensure nginx is running
      service:
        name: nginx
        state: started
        enabled: yes

Modules

Modules are the building blocks used inside tasks — think of them as small programs that perform idempotent actions (install packages, manage files, configure services).

Example (common modules):
- apt: name=git state=present
- copy: src=/tmp/foo dest=/etc/foo owner=root mode=0644
- template: src=site.j2 dest=/etc/site.conf
- service: name=nginx state=started enabled=yes
Learning path
1. Write a small playbook & run with ansible-playbook — 2. Explore modules — 3. Add roles and inventories.
Good to know
Idempotency, inventories, connection methods (ssh), and using --check for dry runs.
